## 1. test-db.js

### Назначение:
**Тестирование и инициализация базы данных**

### Что делает:
1. ✅ Инициализирует БД (создает все таблицы)
2. ✅ Проверяет что таблицы созданы
3. ✅ Тестирует добавление/чтение/удаление данных
4. ✅ Показывает статистику БД

### Когда использовать:
- При первом запуске проекта (создание таблиц)
- Для проверки что БД работает правильно
- Для тестирования подключения

### Запуск:
```bash
npm run init-db
```

### Код разбор:

```javascript
// Импортируем функции и подключение к БД
import { initDatabase } from '../lib/init-db.js';
import db from '../lib/db.js';

// Запускаем инициализацию (создание таблиц)
initDatabase();

// Проверяем что таблицы созданы
const tables = db.prepare(`
  SELECT name FROM sqlite_master 
  WHERE type='table' AND name NOT LIKE 'sqlite_%'
`).all();
// sqlite_master - системная таблица SQLite со списком всех таблиц

// Тест: добавляем тестовый товар
const insertProduct = db.prepare(`
  INSERT INTO products (name, description, price, image_url, active)
  VALUES (?, ?, ?, ?, ?)
`);
// db.prepare() - подготавливает SQL запрос (безопасно от SQL инъекций)
// ? - плейсхолдеры (заменяются значениями при .run())

const result = insertProduct.run(
  'Тестовая футболка',
  'Это тестовый товар',
  1500,
  '/images/test.jpg',
  1
);
// .run() - выполняет INSERT/UPDATE/DELETE

// Читаем обратно
const selectProduct = db.prepare('SELECT * FROM products WHERE id = ?');
const product = selectProduct.get(result.lastInsertRowid);
// .get() - получает одну запись (SELECT)
// lastInsertRowid - ID последней добавленной записи
```

## 2. migrate-add-price-to-variants.js

### Назначение:
**Миграция базы данных** - добавление нового поля `price` в таблицу `product_variants`

### Что такое миграция?
Миграция - это изменение структуры БД (добавление/удаление полей, таблиц).

**Зачем нужны миграции:**
- Версионирование изменений БД
- Можно повторить на другой машине
- Безопасно (проверяет что поле еще не существует)

### Что делает скрипт:
1. ✅ Проверяет существует ли уже поле `price`
2. ✅ Если нет - добавляет поле через `ALTER TABLE`
3. ✅ Показывает информацию о новом поле

### Когда использовать:
- После обновления кода, который требует новое поле
- При работе в команде (чтобы все имели одинаковую структуру БД)
- Один раз при обновлении структуры БД

### Запуск:
```bash
npm run migrate:add-price-to-variants
```

### Код разбор:

```javascript
import db from '../lib/db.js';

// Проверяем структуру таблицы
const tableInfo = db.prepare("PRAGMA table_info(product_variants)").all();
// PRAGMA table_info - показывает информацию о колонках таблицы

// Проверяем есть ли уже поле price
const hasPrice = tableInfo.some(col => col.name === 'price');
// .some() - проверяет есть ли хотя бы один элемент с name === 'price'

if (hasPrice) {
  // Если поле уже есть - выходим
  console.log('Поле уже существует');
  process.exit(0);
}

// Если поля нет - добавляем
db.exec(`
  ALTER TABLE product_variants 
  ADD COLUMN price REAL
`);
// ALTER TABLE - команда SQL для изменения структуры таблицы
// ADD COLUMN - добавляет новую колонку
// db.exec() - выполняет SQL команду напрямую (без параметров)
```

## 3. Удаленный файл: add-products.js

### Что это было:
Скрипт для массового добавления товаров через JavaScript массив.

### Критично ли удаление?
**НЕТ, не критично!** 

**Почему:**
- Вы работаете через SQLite CLI (командная строка)
- Добавляете товары вручную через SQL команды
- Скрипт был опционален

### Что было в файле:
```javascript
const products = [
  {
    name: 'Футболка Pantera - Черная',
    description: '...',
    price: 1500,
    // ...
  }
];

// Цикл который добавлял все товары
for (const product of products) {
  addProduct(product);
}
```

**Преимущества скрипта:**
- ✅ Быстро добавить много товаров
- ✅ Можно хранить данные в коде
- ✅ Удобно для повторного использования

**Преимущества SQLite CLI:**
- ✅ Полный контроль
- ✅ Видно каждый шаг
- ✅ Учитесь SQL
- ✅ Гибкость (можно добавить один товар, обновить, удалить)

## Когда использовать скрипты vs SQLite CLI?

### Используйте скрипты для:
- ✅ Создания таблиц (`test-db.js`)
- ✅ Изменения структуры БД (миграции)
- ✅ Автоматизации повторяющихся задач
- ✅ Массового добавления данных (если много товаров)

### Используйте SQLite CLI для:
- ✅ Ручного добавления товаров
- ✅ Просмотра данных
- ✅ Обновления данных
- ✅ Отладки и проверки
- ✅ Обучения SQL

## Пример: Когда нужен скрипт, а когда CLI

### Сценарий 1: Первый запуск проекта
```bash
# Используем скрипт - создаст все таблицы
npm run init-db
```

### Сценарий 2: Добавление нового поля в БД
```bash
# Используем скрипт миграции
npm run migrate:add-price-to-variants
```

### Сценарий 3: Добавление товара
```bash
# Используем SQLite CLI - полный контроль
sqlite3 database/shop.db
INSERT INTO products ...;
```

### Сценарий 4: Массовое добавление 100 товаров
```bash
# Можно создать скрипт для автоматизации
# Или использовать CLI с SQL файлом
sqlite3 database/shop.db < products.sql
```
