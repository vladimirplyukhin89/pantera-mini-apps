# План реализации Telegram Mini App - Магазин футболок зала бокса

## 1. Описание проекта

Telegram Mini App для продажи футболок зала бокса. Начальная версия поддерживает один тип товара (футболки) с возможностью расширения коллекции в будущем.

## 2. Выбор технологий и базы данных

### Frontend & Backend (unified)
- **Astro** - современный фреймворк для веб-приложений
  - Файловый роутинг (file-based routing)
  - API endpoints через `src/pages/api/`
  - SSR через адаптер Node.js
  - Компоненты `.astro` (HTML-like синтаксис) для статических частей
  - **React** - для интерактивных компонентов (оптимизация DOM-манипуляций)
- **Node.js** (v18+) - runtime для Astro SSR
- **React** - библиотека для создания интерактивных UI компонентов
- **Telegram WebApp SDK** - для интеграции с Telegram


### Управление состоянием
- **React** - для клиентского состояния и интерактивности (useState, useEffect)
- Возможность использовать localStorage для корзины (клиентское состояние)
- React компоненты для оптимизации DOM-манипуляций

### База данных
- **SQLite** - выбрана для упрощения разработки
- **better-sqlite3** - простой драйвер для работы с SQLite
- ✅ Не требует установки отдельного сервера БД
- ✅ Простая настройка и запуск
- ✅ Файловая база данных - легко понять структуру
- ✅ Идеально для небольшого приложения на первом этапе
- ✅ Все данные в одной БД: товары, корзина, заказы
- ✅ Отличная практика SQL на реальных задачах
- ⚠️ В будущем можно мигрировать на PostgreSQL

### Дополнительные инструменты
- **@astrojs/node** - адаптер для SSR на Node.js
- **@astrojs/react** - интеграция React в Astro
- **react** и **react-dom** - React библиотеки
- **dotenv** - управление переменными окружения
- **better-sqlite3** - драйвер для SQLite (все данные: товары, корзина, заказы)


### Почему Astro + React?
**Astro:**
- ✅ Единая структура для frontend и API (не нужно разделять на два проекта)
- ✅ Простой файловый роутинг (страницы в папке `src/pages/`)
- ✅ API endpoints в той же структуре (`src/pages/api/`)
- ✅ Отличная производительность (SSR из коробки)
- ✅ Подходит для SSR приложений (Telegram Mini App)
- ✅ Zero JS по умолчанию (JavaScript загружается только для интерактивных компонентов)

**React:**
- ✅ Оптимизированные DOM-манипуляции (виртуальный DOM, эффективные обновления)
- ✅ Управление состоянием (useState, useEffect)
- ✅ Компонентный подход для интерактивных элементов
- ✅ Интеграция с Astro через `@astrojs/react` (гибридный подход)

**Гибридный подход (Astro + React):**
- ✅ Статические части в `.astro` (быстрая загрузка, SEO-friendly)
- ✅ Интерактивные компоненты в React (оптимизированные DOM-операции)
- ✅ Лучшее из обоих миров: производительность Astro + интерактивность React

**Важно:** 
- Вы будете учиться **Astro**, **React** и **SQLite/SQL** в процессе разработки
- Я буду настраивать основные этапы и структуру, а вы будете продолжать писать код
- Каждый этап включает объяснения, чтобы вы понимали что и зачем делаете

## 4. Архитектура приложения

### 4.1 Архитектура

**Принцип:** Минимум слоев, все данные в одной БД

```
┌─────────────────────────────────────┐
│   Astro Pages (file-based routing)  │  ← Страницы и API endpoints
│   src/pages/                        │
├─────────────────────────────────────┤
│   Astro Components                  │  ← UI компоненты
│   src/components/                   │
├─────────────────────────────────────┤
│   API Endpoints                     │  ← API для товаров, корзины, заказов
│   src/pages/api/                    │
├─────────────────────────────────────┤
│   Database Layer                    │  ← Работа с БД
│   src/lib/db.js (better-sqlite3)   │
├─────────────────────────────────────┤
│   SQLite Database                   │  ← Все данные (товары, корзина, заказы)
└─────────────────────────────────────┘
```

**Особенности архитектуры:**
- **Astro приложение** - единый проект (страницы + API)
- **SQLite** - все данные в одной БД (товары, корзина, заказы)
- Страницы: `src/pages/index.astro` → маршрут `/`
- API endpoints: `src/pages/api/products.ts`, `src/pages/api/cart.ts` и т.д.
- Компоненты: `src/components/ProductCard.astro` → переиспользуемые UI части

**Все данные в SQLite:**
- **Товары** → таблицы `products` и `product_variants` (SQL операции)
- **Корзина и заказы** → таблицы `carts`, `cart_items`, `orders`, `order_items`

**Упрощения:**
- Единый проект (Astro + SQLite)
- Все SQL операции в одном месте (отличная практика SQL)
- Нет зависимости от внешних сервисов
- Прямая работа с БД - полный контроль и понимание
- Данные товаров добавляются напрямую в БД (SQL INSERT) или через seed скрипт

## 5. Схема данных

### Схема базы данных SQLite

**Все данные хранятся в SQLite базе данных**

#### `products`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `name` (TEXT, NOT NULL)
- `description` (TEXT)
- `price` (REAL, NOT NULL)
- `image_url` (TEXT) - путь к изображению товара
- `active` (INTEGER, DEFAULT 1) - 1 = активен, 0 = неактивен
- `created_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- `updated_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)

#### `product_variants`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `product_id` (INTEGER, FK → products.id, NOT NULL)
- `size` (TEXT, NOT NULL) - размер (S, M, L, XL и т.д.)
- `stock` (INTEGER, NOT NULL, DEFAULT 0) - количество на складе
- `created_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- `updated_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- UNIQUE(product_id, size)

**Примечание:** Товары добавляются напрямую в SQLite через SQL INSERT или через seed скрипт. Все данные (товары, корзина, заказы) в одной БД.

#### `carts`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `telegram_user_id` (TEXT, NOT NULL, UNIQUE)
- `created_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- `updated_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)

#### `cart_items`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `cart_id` (INTEGER, FK → carts.id, NOT NULL)
- `product_id` (INTEGER, FK → products.id, NOT NULL)
- `variant_id` (INTEGER, FK → product_variants.id, NOT NULL)
- `size` (TEXT, NOT NULL) - размер товара (для удобства)
- `quantity` (INTEGER, NOT NULL, DEFAULT 1)
- `created_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- UNIQUE(cart_id, variant_id)

#### `orders`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `telegram_user_id` (TEXT, NOT NULL)
- `order_number` (TEXT, UNIQUE) - уникальный номер заказа
- `total_amount` (REAL, NOT NULL)
- `status` (TEXT, DEFAULT 'pending') - pending, confirmed, shipped, delivered, cancelled
- `customer_name` (TEXT, NOT NULL)
- `customer_phone` (TEXT, NOT NULL)
- `customer_address` (TEXT)
- `created_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)
- `updated_at` (DATETIME, DEFAULT CURRENT_TIMESTAMP)

#### `order_items`
- `id` (PK, INTEGER, AUTO_INCREMENT)
- `order_id` (INTEGER, FK → orders.id)
- `product_id` (INTEGER, FK → products.id)
- `product_name` (TEXT) - снимок названия на момент заказа
- `size` (TEXT)
- `quantity` (INTEGER, NOT NULL)
- `price` (REAL, NOT NULL) - снимок цены на момент заказа

## 6. API Endpoints

### Products
- `GET /api/products` - получить все активные товары
- `GET /api/products/:id` - получить товар по ID
- `GET /api/products/:id/variants` - получить варианты товара (размеры)

### Cart
- `GET /api/cart?telegram_user_id=xxx` - получить корзину пользователя
- `POST /api/cart/items` - добавить товар в корзину
- `PUT /api/cart/items/:id` - изменить количество товара
- `DELETE /api/cart/items/:id` - удалить товар из корзины
- `DELETE /api/cart` - очистить корзину

### Orders
- `POST /api/orders` - создать заказ
- `GET /api/orders?telegram_user_id=xxx` - получить заказы пользователя
- `GET /api/orders/:id` - получить заказ по ID

### Contacts
- Статический контент на странице `/contacts` (без API, можно добавить позже)

## 8. Важные замечания

### Упрощения для первого этапа
- ✅ Минимум слоев архитектуры (без overcode)
- ✅ Все данные в одной БД (SQLite) - товары, корзина, заказы
- ✅ Товары добавляются напрямую в SQLite (SQL INSERT) или через seed скрипт
- ✅ Простая работа с БД (better-sqlite3 для обучения SQL)
- ✅ Валидация - простые проверки в API endpoints
- ✅ Нет сложных библиотек для состояния (простой JavaScript)
- ✅ Базовая обработка ошибок (try/catch, без сложных механизмов)

### Особенности Astro для Telegram Mini App
- Astro поддерживает SSR через адаптер Node.js
- API endpoints работают на сервере
- Компоненты рендерятся на сервере (лучше для SEO и производительности)
- Client-side JavaScript добавляется через директивы (client:load, client:idle)
- Файловый роутинг упрощает структуру проекта

### Обучение в процессе
**Важно понимать:**
- Каждый этап включает объяснения
- Вы будете учиться Astro, SQL и работе с БД постепенно
- Не стесняйтесь задавать вопросы в процессе
- Начинаем с простого и постепенно усложняем

### Безопасность (базовый уровень)
- Простая валидация обязательных полей
- Использование prepared statements (better-sqlite3) для защиты от SQL инъекций
- Проверка наличия товара на складе перед добавлением в корзину
- ⚠️ Rate limiting и другие продвинутые меры - в будущем при необходимости

### Идентификация пользователя
- Использование `telegram_user_id` из Telegram WebApp SDK
- Передача `telegram_user_id` в заголовках или query параметрах (без сложной аутентификации)

### Масштабируемость
- Структура БД позволяет легко добавлять новые товары (SQL INSERT в products и product_variants)
- Astro легко масштабируется
- Структура БД легко расширяется (можно добавить категории, промо-акции и т.д.)
- В будущем можно:
  - Мигрировать с SQLite на PostgreSQL
  - Добавить админ-панель для управления товарами
  - Добавить Prisma для более удобной работы с БД (после изучения SQL)
  - Добавить полноценную валидацию (Zod)
  - Добавить библиотеку для состояния (если понадобится)
  - Добавить аутентификацию и авторизацию
  - Использовать TypeScript для типизации (Astro поддерживает .ts файлы)

### Рекомендации по изучению
1. **SQLite и SQL:** Начнем с простых запросов, постепенно усложняем
2. **Astro:** Изучаем файловый роутинг, компоненты, API endpoints по мере необходимости
3. **База данных:** Начнем с better-sqlite3 для понимания SQL, затем можно перейти на Prisma
4. **JavaScript/TypeScript:** Можно использовать .js или .ts файлы (TypeScript опционален)
