### Этап 2: Рефакторинг cart.astro → React компоненты

#### Структура компонентов:

```
src/components/cart/
├── CartPage.tsx          # Главный компонент страницы корзины
├── CartItem.tsx          # Компонент одного товара в корзине
```

#### Преимущества:
- ✅ Централизованное управление состоянием
- ✅ Переиспользуемые компоненты
- ✅ Легче тестировать
- ✅ Меньше проблем с DOM

#### Что будет в cart.astro:
```astro
---
// Только SSR часть - получение данных если нужно
---
<Layout>
  <Header />
  <CartPage client:load />
  <BottomBar client:load />
</Layout>
```

### Этап 3: Рефакторинг product/[id].astro → React компоненты

#### Структура компонентов:

```
src/components/product/

└── useProduct.ts         # Custom hook для логики товара
```

#### Преимущества:
- ✅ Изолированная логика выбора варианта
- ✅ Управление состоянием кнопки
- ✅ Переиспользование компонентов

#### Что будет в product/[id].astro:
```astro
---
// SSR часть - получение товара
const product = getProductById(id);
---
<Layout>
  <Header />
  <ProductPage product={product} client:load />
  <BottomBar client:load />
</Layout>
```

## Детальный план реализации

### Шаг 1: Создать Custom Hooks

**useCart.ts:**
```typescript
- loadCart()
- updateQuantity(itemId, quantity)
- deleteItem(itemId)
- calculateTotal()
- состояние: cart, loading, error
```

**useProduct.ts:**
```typescript
- selectVariant(variant)
- addToCart(productId, variantId)
- состояние: selectedVariant, isAdding
```

**useTelegramUser.ts:**
```typescript
- getTelegramUserId()
- состояние: userId
```

### Шаг 2: Создать компоненты

**CartItem.tsx:**
- Отображение товара
- Кнопки +/-
- Кнопка удаления
- Использует useCart для обновления

**CartSummary.tsx:**
- Подсчет суммы
- Кнопка "Оформить заказ"
- Использует useCart для получения total

**VariantSelector.tsx:**
- Список вариантов
- Выбор варианта
- Использует useProduct для управления выбором

**AddToCartButton.tsx:**
- Кнопка с состояниями (disabled, loading, success)
- Использует useProduct для добавления

### Шаг 3: Интеграция с Astro

- Astro страницы остаются для SSR и layout
- React компоненты для интерактивности
- Использование `client:load` для гидратации
